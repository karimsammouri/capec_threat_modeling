LLM-Based CAPEC Threat Modeling Agent

This agent utilizes a Large Language Model (LLM) and a vector database containing Common Attack Pattern Enumeration and Classification (CAPEC) entries to automate threat modeling. Though this agent is not a substitute for comprehensive threat modeling, it attempts to (1) make the practice more accessible for those without specialized cybersecurity expertise and (2) automate initial, high-level threat modeling, potentially reducing the time, effort, and manual work required. To start, describe your system below.

In this stage, the LLM ‚Äì OpenAI's GPT-4o ‚Äì decomposes your system into its core components using Data-Flow Diagram (DFD) elements: (1) external entities, (2) processes, (3) data stores, and (4) data flows. DFDs are widely used in threat modeling because they highlight how data moves within a system. External entities interact with the system, processes handle internal logic, data stores contain static data, and data flows represent data movement between components. This stage lays the foundation for the threat model and ensures all system elements are analyzed in subsequent stages.

                Using the prior system decomposition, the LLM identifies relevant threats for each system component through its knowledge and 
                reasoning capabilities. Traditionally, this step requires cybersecurity expertise and is done manually, but here it's automated by the LLM.
                The output of this stage serves as a precursor to the subsequent, more detailed CAPEC retrieval stage.
            

            

            
            

            
        


        
        

            
üìù Stage 3: CAPEC Retrieval

            

                For thoroughness and detail ‚Äì and to address LLM limitations like outdated knowledge, hallucinations, and generic responses ‚Äì 
                a retriever maps the threats previously identified by the LLM to the most relevant CAPEC entries using semantic search on the vector database. CAPEC offers regularly updated, industry-standard information with detailed 
                threat descriptions and mitigations. By combining the LLM's reasoning capabilities with CAPEC semantic retrieval, this approach produces a more accurate and comprehensive threat model.
